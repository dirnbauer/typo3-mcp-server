Authorization header issue detected

The MCP endpoint cannot receive Authorization headers. OAuth and bearer-token clients may fail.

Common causes and solutions:

  • Apache: Add to your .htaccess: 
    RewriteEngine On
RewriteCond %{HTTP:Authorization} ^(.*)
RewriteRule .* - [e=HTTP_AUTHORIZATION:%1]
  • HTTP Basic Auth: May conflict with OAuth headers. Prefer IP restrictions if possible.
  • Proxy or CDN: Check whether Authorization headers are stripped upstream.

One-click install in Cursor

Cursor supports remote MCP servers directly. The install button above opens Cursor and offers to add this TYPO3 server automatically.

  1. Click One-click install in Cursor.
  2. Confirm the install in Cursor.
  3. If prompted, complete the OAuth login in TYPO3.
  4. Use the server in Cursor Chat.

Claude Desktop can connect directly to the remote MCP endpoint.

  1. Open Settings → Integrations in Claude Desktop.
  2. Choose Add Integration.
  3. Use {siteName} as the name.
  4. Paste {endpointUrl} as the server URL.
  5. Finish the OAuth flow when Claude Desktop opens TYPO3.
No manual token needed. Claude Desktop can use the remote MCP server directly with OAuth. If you prefer the CLI workflow, use the command above.
Token active Last raw token value cannot be shown again later. Token needed Create a token here, then paste it into your n8n credential.
  1. Add an MCP Client node in n8n.
  2. Set Endpoint to {endpointUrl}.
  3. Set Server Transport to HTTP Streamable.
  4. Use Bearer Auth.
  5. Create the token here and paste it into the n8n credential immediately.
Token active Create a fresh token here if you need to reconnect Manus. Token needed Manus uses a bearer token with the MCP URL.
  1. In Manus, add a new MCP connector.
  2. Use {siteName} as the name.
  3. Set the transport to HTTP.
  4. Use {endpointUrl} as the server URL.
  5. Add an Authorization header with the value Bearer YOUR_CREATED_TOKEN.

MCP Inspector is great for verifying authentication and testing tools interactively.

  1. Run the command in your terminal.
  2. Open the Inspector UI in your browser.
  3. Connect and complete OAuth in TYPO3 if asked.
  4. Call tools and inspect the responses.

For other MCP clients, start with the standard remote MCP endpoint.

  1. Use {endpointUrl} as the server URL.
  2. Let the client discover OAuth automatically via the well-known endpoints.
  3. Use the integration name {siteName} if your client asks for one.
  4. If the client cannot do OAuth, ask an administrator to enable the advanced connection options below.
